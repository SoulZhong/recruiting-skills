# 能力维度参考（16 项）

不作为固定框架——能力和经验揉进优势 / 不足 / 风险的具体描述中。

## 底层能力（6）

### 钻研精神

遇到困难是放弃还是死磕？有没有主动深挖问题的行为？

强信号：能讲清问题背景 → 方法选择 → 局限性；会复盘失败案例；对一个问题能讲出多层细节。

弱信号：第一层追问就讲不清；不主动展开细节；把责任推给"团队 / 数据 / 时间"。

### 学习速度

面对新领域能多快建立认知框架？有没有主动做功课的证据？

强信号：能讲出最近一次"从 0 到能用"的学习路径；主动看源码 / 跑 demo / 写笔记。

弱信号：对新技术只停留在"听说过 / 看到过"；问最近学了什么答不上来。

### 逻辑性（最可靠）

追问时逻辑链是否能延续？能否有条理地拆解问题？

强信号：多轮追问能保持一致；能识别问题假设；遇到新场景能在 1-2 分钟内给出拆解。

弱信号：被追问后前后矛盾；答非所问；只能复述已经讲过的内容。

### 自我认知

是否清楚自己的边界？能否主动说"这个我不懂"？

强信号：能主动说不足；区分"自己负责的"和"团队配合的"；能讲出哪里做得不好。

弱信号：什么都说自己做过；回避"哪里做得不够好"；把团队成果默认成个人成果。

### 表达沟通

能否把复杂问题讲清楚？能否接住追问？

强信号：可以把专业内容讲给外行听；追问时不绕、有条理；能用简洁语言概括复杂链路。

弱信号：使用大量术语但讲不清原理；被追问时打太极；语言节奏失控。

> ⚠️ 表达流畅 ≠ 技术深度强。详见 [attribution-errors.md](attribution-errors.md)。

### 自驱力

没有外部压力时会主动做什么？离职期间有无可验证的探索行为？

强信号：离职期间有独立项目 / 开源 / 内容产出；近一年有主动跨界尝试。

弱信号：长期处于"等任务"状态；离职期间无任何可证伪的行为。

## 经验维度（5）

### 领域 context 理解

对目标赛道是否有基础认知？

观察：能讲清行业 / 用户 / 主要玩家 / 核心问题。

### 产品场景匹配度

与当前产品方向的关联程度？

观察：能不能讲出该产品的典型场景、关键挑战、用户价值。

### 工程落地深度

端到端交付能力？

观察：日志、监控、回滚、上线、稳定性、问题定位经验是否扎实。

### 产品 sense

对用户需求的直觉和判断力？

观察：在面对开放问题时，能否快速识别用户 / 业务核心价值；能否主动取舍优先级。

### 技术栈覆盖

技术工具的掌握广度？

观察：宽度（语言、框架、领域）和深度（任一栈能不能讲清原理 / 实现 / 边界）。

## 风险维度（5）

### 稳定性

离职动机、职场脱节、过往跳动频率。

观察：每段经历是否有合理离职理由；近期是否长时间脱节；自述与简历事实是否一致。

### 技术边界

清楚自己会什么、不会什么？

观察：能否主动承认"这个不熟悉"；面对新概念是否区分"了解过 vs 用过 vs 改造过"。

### 方向偏离

能力结构与岗位核心挑战不匹配。

观察：候选人长项与岗位真正的核心问题是否一致；强能力是否对口；学历 / 经历背景的偏置。

### 团队适配

节奏、文化、协作模式。

观察：偏好独立 vs 团队；适应高压冲刺 vs 稳定流程；与当前团队阶段（冲刺 / 建设 / 维护）的匹配。

### 意愿真实性

对岗位 / 公司 / 方向是否有真实的兴趣和意愿？

观察：是否提前了解公司 / 产品；问"为什么投这个岗位"时是否有具体答案；离职原因是否成熟。
