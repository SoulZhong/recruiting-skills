# 可复用业务定位模板

用户未提供官方口径时，用下列模板作为起点。**根据具体公司和产品替换方括号内容**，不要直接照搬。

## 技术产品 / 平台型

> 我们正在建设一款面向 [目标用户 / 业务场景] 的 [产品 / 平台类型]，通过 [核心技术 / 能力] 帮助用户解决 [核心问题]，提升 [效率 / 质量 / 安全 / 决策 / 增长]。

## 企业服务 / SaaS 型

> 我们为 [目标客户] 提供 [产品 / 服务类型]，覆盖 [关键业务流程]，帮助客户在 [典型场景] 中实现 [业务价值]。

## AI 产品型

> 我们正在建设一款基于 AI 的 [产品类型]，结合 [大模型 / Agent / 数据分析 / 自动化 / 知识库等能力]，帮助用户完成 [核心任务]，让 [原本困难 / 低效 / 依赖人工的事情] 变得更高效、更可靠、更可规模化。

## 行业解决方案型

> 我们面向 [行业 / 领域] 提供 [解决方案类型]，围绕 [关键业务痛点]，整合 [产品、数据、算法、服务或交付能力]，帮助客户实现 [明确结果]。

## 内容 / 媒体 / 研究型

> 我们围绕 [领域 / 人群 / 议题] 提供 [内容、研究、咨询或信息服务]，通过 [数据、专家、AI、社区或方法论]，帮助用户获得 [洞察、判断、决策支持或传播影响力]。

## 使用要点

- **优先用官方口径**：如果有官网、产品手册、招聘页或用户提供的官方介绍，优先使用，模板只是兜底。
- **不要写窄**：通用业务不要被强行限定到单一行业、单一客户。比如"AI 决策引擎"不要被写成"政企客户决策方案"，除非用户明确确认。
- **业务定位放进岗位职责开头**：不要单独设"岗位简介"章节，让 JD 结构贴近招聘平台。
- **不夸大**：不要把"目标""规划""愿景"写成"已实现""客户已覆盖"。
